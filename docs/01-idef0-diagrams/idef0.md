### Описание блока A-0

| Аспект | Содержание |
|---|---|
| **Название** | Управление научными публикациями |
| **Входы** | HTTP-запросы пользователей, данные форм (регистрация, публикация), файлы (изображение + PDF статьи) |
| **Управление** | Бизнес-правила авторизации по JWT, ролевая модель (publisher/reader), CORS-политика, валидация Zod |
| **Механизмы** | Браузер (React SPA), Node.js + Express (API-сервер), MongoDB (база данных), Multer (файловое хранилище) |
| **Выходы** | Список публикаций с пагинацией, содержимое статьи, JWT в cookie, файлы статей, HTTP-ответы (200/401/500) |

## Описание функциональных блоков A0

### A1 — Аутентификация и регистрация

| Параметр | Описание |
|---|---|
| **Входы** | Данные формы регистрации (fullname, login, password, gender, role), данные формы входа (login, password) |
| **Управление** | Схема валидации Zod, правила уникальности login |
| **Механизмы** | Express Router `/api/session/`, bcrypt (хэширование пароля), JWT |
| **Выходы** | Учётная запись пользователя в MongoDB, JWT-токен в cookie |

**Эндпоинты:**
- `POST /api/session/register` — создание новой учётной записи
- `POST /api/session/auth` — аутентификация (выдача JWT)

---

### A2 — Управление сессией

| Параметр | Описание |
|---|---|
| **Входы** | JWT из cookie, запрос проверки статуса, запрос выхода |
| **Управление** | Срок действия токена, политика HTTP-only cookie |
| **Механизмы** | Middleware `verifySession`, cookie-parser |
| **Выходы** | Статус сессии (авторизован/гость), очищенный cookie при выходе |

**Эндпоинты:**
- `POST /api/session/status` — проверка текущей сессии
- `POST /api/session/logout` — завершение сессии

---

### A3 — Создание публикации

| Параметр | Описание |
|---|---|
| **Входы** | Данные статьи (title, description), файлы (image + PDF), данные автора |
| **Управление** | Проверка роли publisher, валидация полей Zod, ограничение размера файлов |
| **Механизмы** | Multer (multipart/form-data), MongoDB (сохранение документа), файловая система |
| **Выходы** | Новый документ Post в MongoDB, файлы в `file_storage/posts/`, ID публикации |

**Эндпоинты:**
- `POST /api/posts/publish` — публикация новой статьи (с файлами)

---

### A4 — Просмотр публикаций

| Параметр | Описание |
|---|---|
| **Входы** | GET-запросы (с параметрами пагинации, ID статьи) |
| **Управление** | Параметры пагинации (page, limit), фильтрация по автору |
| **Механизмы** | MongoDB (aggregate/find), Express Router |
| **Выходы** | Массив постов с мета-данными пагинации, полные данные отдельной статьи |

**Эндпоинты:**
- `GET /api/posts/explore` — лента статей с пагинацией
- `GET /api/posts/read/:id` — детальный просмотр статьи

---

### A5 — Управление файлами

| Параметр | Описание |
|---|---|
| **Входы** | Загруженные файлы (image/pdf) от Multer, запрос файла по URL |
| **Управление** | Правила хранения (привязка к postId), типы файлов |
| **Механизмы** | `fs/promises` (переименование), `express.static` (отдача файлов) |
| **Выходы** | Файлы в `file_storage/posts/`, прямой URL для скачивания |

## Бизнес-глоссарий (10+ терминов)

| Термин | Определение |
|---|---|
| **Публикация (Post)** | Научная статья с заголовком, описанием, изображением и PDF |
| **Автор (Publisher)** | Авторизованный пользователь с правом публикации статей |
| **Читатель (Reader)** | Авторизованный пользователь с правом просмотра публикаций |
| **JWT** | JSON Web Token — токен авторизации, хранящийся в HTTP-only cookie |
| **Сессия** | Авторизованное состояние пользователя, подтверждённое JWT |
| **Пагинация** | Разбиение ленты публикаций на страницы (page + limit) |
| **Мультизагрузка** | Одновременная отправка изображения и PDF через multipart/form-data |
| **Route Guard** | Middleware `verifySession`, защищающий API-эндпоинты от анонимного доступа |
| **Роль** | Тип учётной записи: `p` (publisher) или `r` (reader) |
| **Файловое хранилище** | Директория `file_storage/posts/` для медиафайлов статей |
| **CORS** | Политика разрешения запросов с домена `localhost:5173` к `localhost:3000` |
| **FSD** | Feature-Sliced Design — архитектурный подход к организации React-кода |
