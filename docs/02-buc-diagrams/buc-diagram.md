# BUC — Диаграмма бизнес-прецедентов

> Проект: ScienceJournal — Платформа публикации научных статей  
> Группа: ПИЖ-б-о-24-1 | СКФУ, 2026

---

## Участники (Business Actors)

| Участник | Описание |
|---|---|
| **Гость (Guest)** | Неавторизованный пользователь, имеет доступ только к формам входа/регистрации |
| **Читатель (Reader)** | Авторизованный пользователь с ролью `r` — просматривает статьи |
| **Публицист (Publisher)** | Авторизованный пользователь с ролью `p` — публикует и управляет статьями |
| **Система** | Платформа ScienceJournal (REST API + MongoDB) |

## Описание бизнес-прецедентов

### BUC-1: Получение доступа к платформе

**Инициатор:** Гость  
**Цель:** Получить авторизованный доступ к функционалу платформы

| Шаг | Действие |
|---|---|
| 1 | Гость открывает платформу — отображается экран Welcome |
| 2 | Переход на форму регистрации или входа |
| 3 | Ввод данных (fullname, login, password, gender, role) |
| 4 | Система валидирует данные (Zod), создаёт запись в MongoDB |
| 5 | Система выдаёт JWT в HTTP-only cookie |
| 6 | Пользователь получает доступ к платформе согласно роли |

**Результат:** Авторизованная сессия + роль (p или r)

---

### BUC-2: Потребление контента

**Инициатор:** Читатель, Публицист  
**Цель:** Ознакомиться с публикациями на платформе

| Шаг | Действие |
|---|---|
| 1 | Авторизованный пользователь переходит на страницу Explore |
| 2 | React запрашивает `GET /api/posts/explore?page=1&limit=10` |
| 3 | Сервер возвращает массив постов с пагинацией |
| 4 | Пользователь выбирает статью → переход на `/read/:id` |
| 5 | React запрашивает `GET /api/posts/read/:id` |
| 6 | Отображается заголовок, описание, изображение, ссылка на PDF |
| 7 | Пользователь может скачать PDF напрямую из файлового хранилища |

**Результат:** Прочитанная статья / скачанный PDF

---

### BUC-3: Управление публикациями

**Инициатор:** Публицист  
**Цель:** Опубликовать новую научную статью с медиафайлами

| Шаг | Действие |
|---|---|
| 1 | Публицист переходит на страницу Publish |
| 2 | Заполняет форму: название, описание |
| 3 | Прикрепляет изображение (обложка) и PDF файл статьи |
| 4 | Отправляет `POST /api/posts/publish` (multipart/form-data) |
| 5 | Middleware `verifySession` проверяет JWT |
| 6 | Multer сохраняет файлы во временную директорию |
| 7 | PostService создаёт документ в MongoDB, переносит файлы в `file_storage/posts/` |
| 8 | Клиент получает подтверждение публикации через ModalContext |

**Результат:** Новая публикация доступна в ленте Explore

## Матрица стейкхолдеров

| Стейкхолдер | Роль | Интересы | Влияние |
|---|---|---|---|
| Студент-разработчик | Автор проекта | Качество кода, оценка, портфолио | Высокое |
| Руководитель (Самойлов Ф.В.) | Проверяющий | Соответствие методичке, полнота документации | Высокое |
| Публицисты (конечные пользователи) | Создатели контента | Удобная форма публикации, надёжное хранение файлов | Среднее |
| Читатели (конечные пользователи) | Потребители контента | Быстрый поиск, читабельный UI | Среднее |
| Инфраструктура (MongoDB Atlas / local) | Поставщик данных | Стабильность, безопасность данных | Низкое |
