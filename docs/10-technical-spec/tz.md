## 2. Назначение и цели системы

### 2.1. Назначение

ScienceJournal предназначен для организации публикации, хранения и просмотра научных статей в веб-формате. Система обеспечивает разграничение ролей между публицистами (авторами) и читателями.

### 2.2. Цели

1. Предоставить площадку для публикации научных материалов с поддержкой загрузки PDF.
2. Обеспечить удобный просмотр ленты публикаций с пагинацией.
3. Реализовать безопасную авторизацию на основе JWT.
4. Продемонстрировать полный стек клиент-серверного SPA-приложения.

## 4. Требования к интерфейсу

### 4.1. Веб-интерфейс

| Требование | Описание |
|---|---|
| SPA-навигация | Смена страниц без перезагрузки браузера (React Router) |
| Адаптивность | Корректное отображение на экранах от 320px |

## 5. Требования к технологическому стеку

### 5.1. Backend

- **Runtime:** Node.js ≥ 18
- **Framework:** Express ≥ 4
- **Database:** MongoDB ≥ 6 (с Mongoose или native driver)
- **Auth:** jsonwebtoken (JWT), cookie-parser
- **Validation:** Zod ≥ 3
- **Upload:** Multer

### 5.2. Frontend

- **Framework:** React ≥ 18 (JSX)
- **Build tool:** Vite ≥ 5
- **Routing:** React Router DOM ≥ 6
- **HTTP:** Axios ≥ 1
- **State:** React Context API

### 5.3. API

- Архитектурный стиль: REST
- Формат данных: JSON
- Аутентификация: JWT Bearer / Cookie
- Минимум эндпоинтов: 8
