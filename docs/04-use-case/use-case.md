# Use Case — Диаграмма и спецификации прецедентов

> Проект: ScienceJournal — Платформа публикации научных статей  
> Группа: ПИЖ-б-о-24-1 | СКФУ, 2026

---

## Акторы системы

| Актор | Описание |
|---|---|
| **Гость** | Не авторизован, видит только Welcome и страницы Auth/Register |
| **Читатель (Reader)** | Авторизован с ролью `r`, может просматривать ленту и читать статьи |
| **Публицист (Publisher)** | Авторизован с ролью `p`, может всё что Читатель + публиковать статьи |

## Спецификации прецедентов

### UC1 — Регистрация

| Поле | Значение |
|---|---|
| **ID** | UC1 |
| **Название** | Регистрация нового пользователя |
| **Актор** | Гость |
| **Предусловие** | Пользователь не авторизован, открыта страница Register |
| **Основной сценарий** | 1. Ввод fullname, login, password, gender, role → 2. `POST /api/session/register` → 3. Zod-валидация → 4. Хэширование пароля → 5. Сохранение в MongoDB → 6. JWT в cookie → 7. Редирект на Explore |
| **Альтернативный сценарий** | Логин уже занят → MongoDB Duplicate Key → HTTP 409 → уведомление в UI |
| **Постусловие** | Учётная запись создана, пользователь авторизован |

---

### UC2 — Аутентификация

| Поле | Значение |
|---|---|
| **ID** | UC2 |
| **Название** | Вход в систему |
| **Актор** | Гость (с существующим аккаунтом) |
| **Предусловие** | Учётная запись существует в MongoDB |
| **Основной сценарий** | 1. Ввод login, password → 2. `POST /api/session/auth` → 3. Поиск User по login → 4. Сравнение password → 5. JWT в cookie → 6. Редирект на Explore |
| **Альтернативный сценарий** | Неверный пароль → HTTP 401 → уведомление "Неверные данные" |
| **Постусловие** | JWT-сессия активна, пользователь авторизован согласно роли |

---

### UC3 — Просмотр ленты статей

| Поле | Значение |
|---|---|
| **ID** | UC3 |
| **Название** | Просмотр ленты публикаций |
| **Актор** | Читатель, Публицист |
| **Предусловие** | Пользователь авторизован |
| **Основной сценарий** | 1. Открыть страницу Explore → 2. `GET /api/posts/explore?page=1&limit=10` → 3. Middleware `verifySession` → 4. MongoDB: find с пагинацией → 5. Отображение карточек Post |
| **Постусловие** | Список публикаций загружен |

---

### UC4 — Чтение статьи

| Поле | Значение |
|---|---|
| **ID** | UC4 |
| **Название** | Просмотр полной публикации |
| **Актор** | Читатель, Публицист |
| **Предусловие** | Пользователь авторизован, выбрана публикация из ленты |
| **Основной сценарий** | 1. Клик на карточку → 2. Переход `/read/:id` → 3. `GET /api/posts/read/:id` → 4. MongoDB: findById → 5. Отображение title, description, authorFullName, image → 6. Кнопка «Скачать PDF» открывает `/file_storage/posts/{id}.pdf` |
| **Постусловие** | Статья прочитана, PDF доступен |

---

### UC5 — Выход из системы

| Поле | Значение |
|---|---|
| **ID** | UC5 |
| **Название** | Завершение сессии |
| **Актор** | Читатель, Публицист |
| **Предусловие** | Пользователь авторизован |
| **Основной сценарий** | 1. Клик «Выйти» в Header → 2. `POST /api/session/logout` → 3. Сервер очищает cookie → 4. Клиент сбрасывает ProfileContext → 5. Редирект на Auth |
| **Постусловие** | JWT-сессия завершена, пользователь — гость |

---

### UC6 — Публикация статьи

| Поле | Значение |
|---|---|
| **ID** | UC6 |
| **Название** | Создание новой публикации |
| **Актор** | Публицист |
| **Предусловие** | Авторизован с ролью `p`, открыта страница Publish |
| **Включает** | UC7 (Загрузка файлов) |
| **Основной сценарий** | 1. Заполнение формы (title, description) → 2. Прикрепление image + pdf → 3. `POST /api/posts/publish` (multipart) → 4. verifySession → 5. Multer сохраняет файлы → 6. PostService создаёт документ → 7. Файлы перемещаются в `file_storage/posts/` → 8. Уведомление об успехе |
| **Альтернативный сценарий** | Нет прав (роль `r`) → HTTP 403 → уведомление об ошибке |
| **Постусловие** | Публикация создана и доступна в ленте Explore |

---

### UC7 — Загрузка файлов (включаемый прецедент)

| Поле | Значение |
|---|---|
| **ID** | UC7 |
| **Название** | Загрузка медиафайлов публикации |
| **Актор** | Система (Multer) |
| **Включается в** | UC6 |
| **Основной сценарий** | 1. Multer принимает multipart/form-data → 2. Сохранение в `uploads/` (временно) → 3. PostService переименовывает файлы по `postId` → 4. Перемещение в `file_storage/posts/` |
| **Постусловие** | Файлы доступны по URL `/file_storage/posts/{id}.{ext}` |
